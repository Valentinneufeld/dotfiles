# Session-Notizen — 2026-02-10

## Thema: Session-Persistenz & Terminal-Multiplexer

### 1. WezTerm Session-Persistenz

**Frage:** Kann WezTerm Sessions nach einem PC-Neustart wiederherstellen?

**Antwort:** Nein. WezTerm hat keine eingebaute Session-Persistenz. Workspaces, Tabs, Panes und deren Inhalte leben nur im Arbeitsspeicher und gehen bei einem Neustart verloren.

**Workarounds in WezTerm:**
- `gui-startup` Event: Festes Layout beim Start erzeugen (Tabs, Splits), aber kein Scrollback/Prozess-Restore
- `wezterm cli` Skripte: Tabs/Panes automatisiert öffnen und Befehle starten

### 2. Terminals mit (Teil-)Restore

Kein Terminal-Emulator kann laufende Prozesse und Scrollback nach einem Reboot wiederherstellen — das ist eine OS-Limitierung.

| Terminal | Was wird restored | Plattform |
|----------|------------------|-----------|
| **Windows Terminal** | Tabs, Profile, Working Directories | Windows |
| **iTerm2** | Tabs, Splits, Working Directories, teilweise Scrollback | macOS |
| **Kitty** | Sessions manuell speichern/laden (Layouts) | Linux/macOS |
| **Konsole** | Profile und Layouts | Linux (KDE) |

**Fazit:** Session-Persistenz ist das Problem eines Terminal-Multiplexers (tmux, Zellij), nicht des Emulators.

### 3. Zellij — Moderner Terminal-Multiplexer

Zellij ist eine in Rust geschriebene Alternative zu tmux.

**Killer-Feature: Session Resurrection**
- Sessions werden **automatisch** persistiert (Pane-Struktur, Arbeitsverzeichnisse, laufende Befehle)
- `zellij attach` stellt laufende Sessions wieder her
- Funktioniert auch nach Reboot — **kein Plugin nötig** (vs. tmux-resurrect + tmux-continuum)

**Weitere Features:**
| Feature | Details |
|---------|---------|
| UI out of the box | Statusleiste zeigt verfügbare Keybindings direkt an |
| Floating Panes | Schwebende Fenster über anderen Panes |
| Stacked Panes | Panes übereinander stapeln, per Tab wechseln |
| Layouts | Deklarative YAML-Layouts für reproduzierbare Workspaces |
| Plugin-System | WebAssembly-Plugins — in jeder Sprache die zu WASM kompiliert |
| Web-Client | Sessions im Browser anzeigen (Pair Programming) |
| tmux-Mode | Eingebauter Kompatibilitätsmodus mit tmux-Keybindings |

**Zellij vs tmux:**

| | Zellij | tmux |
|---|--------|------|
| Sprache | Rust | C |
| Session-Restore | eingebaut | Plugin (resurrect) |
| Lernkurve | flach (UI zeigt Bindings) | steil |
| Plugin-System | WebAssembly | Shell-basiert |
| Ökosystem | jung, wachsend | riesig, bewährt |
| Stabilität | gut, aber jünger | battle-tested seit 2007 |

**Installation in WSL:**
```bash
cargo install zellij
# oder
bash <(curl -L zellij.dev/launch)
```

**Grundlegende Nutzung:**
```bash
zellij                          # Neue Session starten
zellij attach                   # Letzte Session wiederherstellen
zellij attach session-name      # Bestimmte Session
zellij list-sessions            # Alle Sessions auflisten
```

**Kombination mit WezTerm:** Zellij könnte das Pane/Tab/Session-Management übernehmen, WezTerm nur als "dummer" Emulator (Font, Farben, GPU-Rendering). Aktuelles WezTerm-Setup ist aber schon sehr ausgebaut — Umstieg wäre Aufwand, Hauptgewinn wäre die Session Resurrection.
